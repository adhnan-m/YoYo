# Amazon Affiliate Website - Implementation Plan

Build a production-ready Amazon affiliate website with admin panel using Next.js 14+ (App Router), TailwindCSS, MongoDB Atlas, NextAuth.js, and Cloudinary. Designed for Vercel deployment via GitHub.

## Proposed Changes

### Phase 1: Project Scaffolding

#### [NEW] Project initialization
- `npx create-next-app@latest ./` with TypeScript, TailwindCSS, App Router, ESLint
- Install deps: `mongoose`, `next-auth`, `bcryptjs`, `zod`, `cloudinary`, `next-cloudinary`, `react-icons`, `slugify`

#### [NEW] `.env.example`
Template for all required env vars (MongoDB URI, NextAuth secret, Cloudinary keys, etc.)

---

### Phase 2: Database & Models

#### [NEW] `lib/mongodb.ts`
Cached MongoDB connection using Mongoose (serverless-safe pattern).

#### [NEW] `models/Product.ts`
Fields: name, slug, description, categoryId, price, imageUrl, affiliateUrl, featured, clicks, createdAt.

#### [NEW] `models/Category.ts`
Fields: name, slug, createdAt.

#### [NEW] `models/Subscriber.ts`
Fields: email, createdAt.

#### [NEW] `models/Admin.ts`
Fields: email, password (hashed), role, createdAt.

## Verification Plan

### Automated Tests
- Run `npm run build` to verify zero build errors and all pages compile
- Run the seed script to verify DB models and connections work

### Manual Verification (Browser)
1. Start dev server (`npm run dev`), open http://localhost:3000
2. Homepage: Verify hero section, featured products, categories, newsletter form renders
3. Product page: Click a product -> verify slug page loads with image, price, affiliate button
4. Category page: Navigate to category -> verify product list, search, pagination
5. Admin login: Go to /admin/login -> login with seeded admin credentials
6. Admin dashboard: Verify stats cards display
7. Product CRUD: Create, edit, delete a product
8. Dark/light mode: Toggle theme and verify styling
9. Mobile: Resize browser to verify responsive layout

NOTE: Cloudinary upload requires valid API keys. Image upload will be tested manually after env vars are configured.
